# Licenta-Completa

RAMY Procedural Engine manages to generate 3D assets for game developers or artists by using a simple yet powerful node-based system with a generation stack based on components.

The point is that you can generate any kind of scene with multiple gameobjects, those objects have components that can be modified and stitched together.

## DESCRIPTION:
The RAMY Procedural Generation Studio is a modular, extensible framework designed for the automatic generation of three-dimensional content. The system architecture follows a strict layered approach, separating concerns across five distinct architectural layers that communicate through well-defined interfaces. This separation ensures maintainability, testability, and allows for seamless extension of the system's capabilities without modifying the core engine.
## Presentation Layer
The Presentation Layer serves as the primary interface between the user and the underlying system. At its core sits the Main Window component, which orchestrates all user interface elements and manages the application's visual state. The 3D Viewport is the central visual component, providing real-time hardware-accelerated rendering of the generated scene through OpenGL. Users can navigate the viewport using standard camera controls including orbit, pan, and zoom operations, with a reference grid displayed for spatial orientation.
The Pipeline Manager presents the generation stack as a vertical list of generator cards. Each card represents a single step in the procedural generation pipeline, and users can reorder these steps through drag-and-drop interactions. This visual representation allows artists and developers to understand and modify the generation sequence intuitively. The Inspector Panel is a context-sensitive component that dynamically adapts its contents based on the currently selected generator, displaying relevant parameters as sliders, text fields, or dropdown menus depending on the data type. Finally, the Asset Explorer provides a gallery-style interface for browsing previously generated and saved assets, with filtering capabilities by name, date, and custom tags.
## Core Engine Layer
The Core Engine represents the computational heart of the application. The Application class implements the Singleton pattern, ensuring a single point of entry and global access to the engine's subsystems. It manages the initialization sequence, the main update loop, and the graceful shutdown of all resources.
The Scene Manager is responsible for maintaining the hierarchical structure of all objects within the 3D scene. It stores and manages a collection of GameObject instances, each representing a discrete entity in the virtual world. Every GameObject is composed of a Transform component (storing position, rotation, and scale as Vector3 and Quaternion data) and a Mesh component (containing vertex data, indices, and calculated normals). The Scene Manager provides methods for loading and saving scene state, as well as adding, removing, and querying objects.
The Generation Stack implements a pipeline execution model where generators are stacked and executed sequentially. Each generator in the stack receives the output of the previous generator as input, allowing for incremental transformations. For example, a typical pipeline might first generate base terrain using noise, then apply erosion simulation, followed by vegetation scattering, and finally structure placement. The Renderer component handles all OpenGL operations, managing shader programs, texture units, framebuffer objects for shadow mapping, and the final composite pass that displays the scene to the viewport.
## Procedural Generators Layer
The Procedural Generators layer embodies the plugin-based architecture that makes the system extensible. At the center of this layer is the IGenerator interface, which defines the contract that all generator implementations must fulfill. This interface specifies three essential methods: Execute() which performs the actual generation algorithm, GetParameters() which returns a dictionary of configurable parameters exposed to the Inspector Panel, and GetResult() which returns the generated mesh data.
The system ships with four demonstration implementations of this interface. The Perlin Noise Generator produces coherent noise patterns suitable for terrain heightmaps and organic surface variations, with configurable frequency, amplitude, and octave parameters. The Wave Function Collapse (WFC) Generator implements the constraint-based algorithm for generating structures that adhere to adjacency rules, commonly used for dungeon generation or tile-based environments. The Shape Grammar Generator applies formal grammar rules to produce architectural structures procedurally, allowing for the creation of buildings, bridges, and other man-made constructs. The Scatter Generator handles the pseudo-random distribution of objects across surfaces using Poisson disc sampling to ensure natural-looking placement of vegetation, rocks, or decorative elements.
The plugin architecture means developers can implement additional generators by simply creating new classes that implement the IGenerator interface. These can be loaded dynamically at runtime, enabling hot-swapping of generation algorithms without recompiling the core engine.
